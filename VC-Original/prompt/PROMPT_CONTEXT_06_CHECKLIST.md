# ✅ 세팅 마무리 검수 체크리스트 (Final Check)

새로운 프로젝트나 가이드 컴포넌트 세팅이 끝난 시점(마스터 프롬프트의 모든 단계 완료 후)에 AI는 스스로 이 문서를 읽고 아래 항목들이 의도대로 잘 세팅되었는지 최종 검토해야 합니다. 어긋난 항목이 있다면 즉시 코드를 수정해야 합니다.

## 1. 🎨 스타일링 및 환경 검수
- **SCSS 환경 준수**: `PROMPT_CONTEXT_01_SETTINGS.md` 및 `PROMPT_CONTEXT_05_SCSS.md`에 명시된 대로 Tailwind CSS 개입 없이 100% SCSS Modules 방식으로 컴포넌트 스타일이 작성되었는가? (Tailwind 환경일 경우 보조 규칙으로 무시)
- **CSS 변수 사용**: 컬러, 타이포그래피 등의 하드코딩된 값 대신 `var(--color-primary-500)` 같은 `:root` 토큰 변수가 올바르게 사용되었는가?

## 2. 🧩 컴포넌트 프리뷰 (오픈 렌더링) 강제 검수
- **Card/Border 래핑 제거**: 가이드 페이지(`.preview` 영역) 내에 위치한 컴포넌트들이 억지로 `<Card>` 컴포넌트로 감싸여 있거나, 불필요한 `border`, `shadow`, `bg-white` 등의 박스 테두리가 씌워져 있지 않은가?
- **자연스러운 렌더링**: 컴포넌트 본연의 형태 그대로 화면에 개방형(Open)으로 렌더링되고 있는가?

## 3. 📂 레이아웃 분리 검수
- **가이드 전용 레이아웃**: 가이드 페이지(`/guide/*`)가 전역 사용자 Layout이 아닌, `guide-layout.tsx`로 완벽하게 독립되어 렌더링되고 있는가?
- **대시보드 vs 가이드 문서**: `/guide/dashboard` 화면에서는 사이드바가 숨겨지고 전체 너비를 차지하며, 그 외의 문서 페이지는 사이드바가 존재하고 최대 너비(1024px)로 중앙 정렬되어 있는가?

## 4. 🚀 생성 과정 검수
- **컴포넌트 일괄 생성 누락 확인**: 마스터 프롬프트의 지시대로 컴포넌트들이 중간에 멈추거나 누락되지 않고 전체가 한 번에 정상적으로 추가되었는가?
- **사이드바 카테고리 동작**: 디자인 시스템 좌측 메뉴(Sidebar)의 카테고리(Foundations, Elements 등)별 접고 펼침(Accordion/Toggle) 동작이 정상적으로 구현되어 기능하고 있는가?
